# HH Job Assistant

Chrome-расширение для hh.ru: помогает запускать отклики, поднимать резюме, обрабатывать чаты и заполнять сложные формы через Groq.

Расширение работает только в профиле Chrome/Chromium, где пользователь уже вошел в hh.ru.

## Возможности

- Показывает статус в компактном всплывающем окне: версию, кнопку настроек, готовность hh.ru, состояние Groq, текущее действие, счетчики, последние действия и отчеты по чатам.
- Запускает отклики со страницы поиска вакансий hh.ru.
- Без ключа Groq API продолжает обычные отклики, но пропускает вакансии, где нужны сопроводительные письма, вопросы работодателя или тесты.
- С ключом Groq API составляет сопроводительные письма и ответы на вопросы работодателя.
- Останавливает активный процесс кнопкой `Стоп`.
- Поднимает резюме через сохранение/обновление резюме на hh.ru.
- Обрабатывает чаты hh.ru при наличии ключа Groq API: по умолчанию только непрочитанные, в режиме черновика без отправки.
- Фиксирует отчеты по чатам, если работодатель просит перейти во внешний канал связи.
- Позволяет скопировать текст ошибки или пропуска из последних действий.
- Пишет технический лог в локальное хранилище профиля расширения, без отдельной вкладки во всплывающем окне.

## Установка

1. Скачайте репозиторий ZIP-архивом или клонируйте его.
2. Откройте в Chrome `chrome://extensions`.
3. Включите `Режим разработчика`.
4. Нажмите `Загрузить распакованное расширение`.
5. Выберите папку репозитория, где лежит `manifest.json`.
6. Закрепите расширение на панели Chrome.

После обновления файлов нажмите `Перезагрузить` у расширения на `chrome://extensions`.

## Настройка

Откройте настройки через шестеренку во всплывающем окне.

Основные поля:

- Модель Groq: по умолчанию `llama-3.3-70b-versatile`. Доступны `llama-3.3-70b-versatile`, `llama-3.1-8b-instant`, `openai/gpt-oss-120b`, `openai/gpt-oss-20b`.
- Ссылка на резюме hh.ru: URL вида `https://hh.ru/resume/...`.
- Кэш резюме, часов: по умолчанию `1`, допустимый диапазон от `0.1` до `168`.
- Ожидаемая зарплата: используется как честное запасное значение для вопросов о зарплате.
- Промпт для сопроводительного письма: инструкция для Groq, как писать письмо.
- Дневной лимит откликов: `100`.
- Минимальная пауза, мс: `1500`.
- Максимальная пауза, мс: `3000`.
- Обработка чатов: по умолчанию только непрочитанные, режим черновика, лимит `10`.
- Ключ Groq API: нужен для сопроводительных писем, вопросов работодателя, тестов и обработки чатов.
- Кнопка `Проверить Groq`: проверяет ключ и модель без запуска откликов.

Ключ Groq API можно создать в [Groq Console](https://console.groq.com/docs/quickstart).

Секреты не должны попадать в Git. Не коммитьте ключи API, файлы cookie, токены, профили браузера, `.env` и любые файлы с авторизацией.

## Использование

1. Войдите в hh.ru в том же профиле Chrome/Chromium, где установлено расширение.
2. Откройте страницу поиска вакансий: `https://hh.ru/search/vacancy?...`.
3. Откройте всплывающее окно расширения.
4. Проверьте статус:
   - `ГОТОВО`: hh.ru открыт, Groq подключен.
   - `ГОТОВО, без автоответов`: ключа Groq API нет, сложные вакансии будут пропущены, обработка чатов недоступна.
   - Красный статус: всплывающее окно показывает конкретный блокер, например `Откройте hh.ru` или `Войдите в hh.ru`.
5. Нажмите `Запуск откликов`.
6. Для остановки нажмите `Стоп`.
7. Для поднятия резюме нажмите `Поднятие резюме`.
8. Для чатов нажмите `Обработка чатов`.
9. Для очистки списка отчетов по чатам нажмите `Очистить`.

## Поведение без ключа Groq API

Без ключа расширение не падает и не считает это ошибкой.

Оно продолжает простые отклики, но пропускает вакансии, где нужны:

- сопроводительное письмо;
- ответы на вопросы работодателя;
- тестовое задание;
- сложная форма с выбором вариантов.

Причина пропуска отображается в последних действиях и пишется в технический лог.

Обработка чатов без ключа Groq API недоступна, потому что ответ в чат всегда генерируется через Groq.

## Проверка и логи

Юнит-тесты не считаются полноценным тестированием этого расширения. Для приемки нужен запуск в браузере на реальном hh.ru с авторизованным профилем.

Базовые команды:

```bash
npm test
npm run test:extension
npm run test:hh:chromium
```

`npm run test:extension` запускает временный Chromium-профиль и проверяет, что распакованное расширение грузится.

`npm run test:hh:chromium` использует постоянный профиль `.hhja-chromium-profile`. Этот профиль должен быть авторизован на hh.ru. Если профиль не авторизован, откройте браузер для входа:

```bash
HHJA_CHROMIUM_KEEP_OPEN=1 npm run test:hh:chromium
```

После входа повторите:

```bash
npm run test:hh:chromium
```

Для ограниченного запуска через реальное расширение:

```bash
HHJA_LIMIT=1 HHJA_MAX_PROCESSED=1 HHJA_CHROMIUM_RUN_MS=60000 npm run start:hh:chromium
```

`HHJA_MAX_PROCESSED=1` нужен для безопасной приемки: запуск останавливается после одной обработанной вакансии, включая пропуск.

Технический лог хранится локально в хранилище профиля расширения:

- `agentDebugLogFile`: имя текущего лог-артефакта, например `hh-job-assistant-2026-06-17T05-23-45Z.debug`;
- `agentDebugLogText`: текст лога в JSONL-формате;
- `agentDebugLog`: последние структурированные события;
- `runState`: текущее состояние;
- `runResults`: результаты откликов.

Лог запросов Groq хранит метаданные: задачу, модель, число сообщений, длины сообщений, лимит токенов и длину ответа. Полный текст резюме, вакансии, чата, сопроводительного письма и ответа Groq в технический лог не пишется.

Анализ локальных логов:

```bash
npm run inspect:logs -- --storage-dir ".hhja-chromium-profile/Default/Local Extension Settings/<extension-id>"
```

Эта команда запускает `scripts/inspect-extension-log.mjs`.

Если используется обычный профиль Chrome, путь будет вида:

```text
~/Library/Application Support/Google/Chrome/<Profile>/Local Extension Settings/<extension-id>
```

Для проверки hh.ru нужен авторизованный профиль. Имена технических событий в логе служебные; пользовательский интерфейс остается на русском.

При ошибках заполнения форм работодателя не ограничивайтесь логами. Логи показывают этапы работы и размеры запросов, но причину неправильной привязки полей быстрее искать в живом DOM открытой вкладки Chrome. Если в `groq_request_payload` видны `task_...`, пустые варианты или ответы попали не в те поля, сразу инспектируйте структуру формы. Для текущих hh.ru форм надежный контракт: каждый вопрос и его radio/textarea лежат внутри ближайшего `div[data-qa="task-body"]`; сначала используйте этот контейнер, а поиск по тексту страницы оставляйте только как запасной путь.

## Команды для разработки

```bash
npm test
npm run test:extension
npm run test:hh:chromium
npm run inspect:logs -- --json
```

Настройка постоянного Chromium-профиля из `.env`:

```bash
HHJA_ENV_FILE=/path/to/.env npm run configure:hh:chromium
```

Поддерживаемые переменные включают `GROQ_API_KEY`, `GROQ_MODEL`, `HHJA_RESUME_URL`, `HHJA_EXPECTED_SALARY` и другие `HHJA_*` настройки. Скрипт не печатает секретные значения.

Синхронизация файлов cookie hh.ru из Chrome в `.hhja-chromium-profile`:

```bash
npm run sync:hh:auth
```

Используйте это только локально. Файлы cookie и профили браузера нельзя коммитить.

## Ограничения и безопасность

- Расширение не хранит логин или пароль hh.ru.
- Если hh.ru показывает login/captcha/anti-bot, автоматические действия не запускаются.
- Если работодатель просит перейти в Telegram, WhatsApp, email, телефон или внешний сайт, расширение создает отчет и не отправляет автоответ.
- Режим автоотправки в чатах отправляет сгенерированный текст без ручной проверки. Включайте его только осознанно.
- Репозиторий публичный: секреты, файлы cookie, токены и локальные профили должны оставаться только локально.
